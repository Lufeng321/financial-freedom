## 前期准备

### 1. 账号准备

#### 微信小程序账号
- 访问 [微信公众平台](https://mp.weixin.qq.com/)
- 注册小程序账号（个人/企业）
- 完成认证（个人账号无需认证，企业账号需要）
- 获取 AppID

#### 腾讯云账号
- 访问 [腾讯云控制台](https://console.cloud.tencent.com/)
- 注册腾讯云账号
- 实名认证
- 确保账户有足够余额（云开发免费额度）

### 2. 开发工具安装

#### 微信开发者工具
- 下载地址: https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
- 安装并登录微信账号
- 选择稳定版或最新版本

#### Node.js 环境
- 下载地址: https://nodejs.org/
- 推荐版本: Node.js 18.x LTS
- 验证安装: `node -v` 和 `npm -v`

### 3. 代码准备

#### 克隆项目
```bash
git clone <your-repository-url>
cd 个人成长小程序
```

#### 安装依赖
```bash
npm install
```

#### 构建组件库（如使用 TDesign）
```bash
# 如果使用了 npm 组件
npm run build:npm
# 或在微信开发者工具中点击"工具" -> "构建 npm"
```

## 环境配置

### 1. 项目配置文件

#### project.config.json
```json
{
  "appid": "your-appid",
  "projectname": "个人成长小程序",
  "compileType": "miniprogram",
  "cloudfunctionRoot": "cloudfunctions/",
  "cloudbaseRoot": "cloudbase/",
  "setting": {
    "urlCheck": true,
    "es6": true,
    "postcss": true,
    "minified": true
  }
}
```

**重要参数说明**:
- `appid`: 替换为你的小程序 AppID
- `urlCheck`: 生产环境设为 true，开发环境可设为 false
- `es6`: 开启 ES6 转 ES5

#### cloudbaserc.json
```json
{
  "envId": "your-env-id",
  "region": "ap-shanghai",
  "functionRoot": "cloudfunctions"
}
```

**参数说明**:
- `envId`: 云开发环境ID（需要先创建）
- `region`: 地域（ap-shanghai/ap-guangzhou/ap-beijing）

### 2. 云环境配置文件

#### config/cloud-env.js
```javascript
module.exports = {
  envId: 'your-env-id'  // 替换为实际的云开发环境ID
};
```

### 3. 创建云开发环境

#### 步骤一：开通云开发
1. 打开微信开发者工具
2. 点击顶部菜单"云开发"
3. 点击"开通"
4. 选择"免费版"或"基础版"
5. 支付开通费用

#### 步骤二：配置环境信息
1. 记录下 `环境ID`
2. 记录下 `环境名称`
3. 复制到 `cloudbaserc.json` 和 `config/cloud-env.js`

## 云开发部署

### 1. 数据库部署

#### 创建数据库集合
登录云开发控制台 -> 数据库 -> 新建集合

**按顺序创建以下集合**:

1. `user` - 用户信息
2. `bills` - 账单记录
3. `assets` - 资产管理
4. `liabilities` - 负债管理
5. `recurring_bills` - 周期账单
6. `budgets` - 预算管理
7. `skills` - 技能管理
8. `sessions` - 专注会话
9. `ai_plans` - AI学习计划
10. `habits` - 习惯管理
11. `sport_records` - 运动记录
12. `diary_entries` - 反思日记
13. `todos` - 待办任务
14. `goals` - 目标管理
15. `kpi_snapshots` - KPI快照
16. `events` - 事件埋点
17. `weekly_stats` - 每周统计
18. `achievements` - 成就定义
19. `user_achievements` - 用户成就
20. `user_quests` - 每周任务
21. `user_settings` - 用户设置

#### 设置数据库权限
1. 进入每个集合
2. 点击"权限设置"
3. 选择"自定义权限"或"仅创建者可读写"

**权限配置示例**:
```json
{
  "read": "doc.userId == auth.openid",
  "write": "doc.userId == auth.openid"
}
```

**特殊情况**:
- `achievements`: 所有用户可读
  ```json
  {
    "read": true,
    "write": false
  }
  ```

#### 创建数据库索引
为优化查询性能，创建以下索引:

**bills 集合**:
- userId + datetime
- userId + kind

**kpi_snapshots 集合**:
- userId + date (唯一索引)

**events 集合**:
- userId + ts
- userId + sessionId

**其他集合**:
- 所有集合都应有 userId 索引

### 2. 云存储部署

#### 创建存储桶
1. 进入云开发控制台 -> 云存储
2. 点击"新建"
3. 输入存储桶名称（如 `growth-app`）
4. 选择地域（与云环境一致）
5. 点击确定

#### 设置存储权限
- 开发测试: `所有用户可读写`
- 生产环境: `仅创建者可读写` 或 `自定义权限`

#### 创建文件夹结构
```
growth-app/
├── avatars/          # 用户头像
├── diary/            # 日记图片
├── export/           # 导出文件
└── temp/             # 临时文件
```

### 3. 云函数部署

#### 本地安装依赖
每个云函数需要单独安装依赖:

```bash
# 进入云函数目录
cd cloudfunctions/login
npm install

# 依次为每个云函数安装依赖
cd ../events-batch && npm install
cd ../snapshot-daily && npm install
cd ../create-user && npm install
cd ../daily-stats && npm install
cd ../gain-xp && npm install
cd ../achievements-meta && npm install
```

#### 微信开发者工具部署

**方式一：单文件部署**
1. 右键点击云函数文件夹（如 `login`）
2. 选择"上传并部署:云端安装依赖"
3. 等待部署完成

**方式二：批量部署**
1. 右键点击 `cloudfunctions` 目录
2. 选择"上传并部署:云端安装依赖"
3. 选择需要部署的云函数

**方式三：本地调试**
1. 右键点击云函数文件夹
2. 选择"本地调试"
3. 在本地运行和调试

#### 云函数配置
为每个云函数配置参数:

**login 云函数**:
- 内存: 128MB
- 超时: 10秒
- 环境变量: 无

**events-batch 云函数**:
- 内存: 128MB
- 超时: 10秒
- 环境变量: 无

**snapshot-daily 云函数**:
- 内存: 128MB
- 超时: 10秒
- 环境变量: 无

**create-user 云函数**:
- 内存: 128MB
- 超时: 10秒
- 环境变量: 无

**daily-stats 云函数**:
- 内存: 256MB
- 超时: 20秒
- 环境变量: 无

**gain-xp 云函数**:
- 内存: 128MB
- 超时: 10秒
- 环境变量: 无

**achievements-meta 云函数**:
- 内存: 128MB
- 超时: 10秒
- 环境变量: 无

#### 测试云函数
1. 打开云开发控制台 -> 云函数
2. 点击云函数名称
3. 点击"云端测试"
4. 输入测试参数
5. 点击"测试"

**测试示例 - login**:
```json
{}
```

**测试示例 - create-user**:
```json
{
  "nickname": "测试用户",
  "avatarEmoji": "👤"
}
```

### 4. 云函数定时器（可选）

如果需要每日自动执行 `snapshot-daily`:

#### 创建定时触发器
1. 进入云开发控制台 -> 云函数
2. 点击 `snapshot-daily` 函数
3. 点击"触发器" -> "新建"
4. 选择"定时触发器"
5. 设置 cron 表达式: `0 0 * * *` (每天0点执行)
6. 保存

## 小程序发布

### 1. 开发环境测试

#### 本地调试
1. 在微信开发者工具中点击"编译"
2. 使用真机扫码调试
3. 测试核心功能:
   - 用户登录
   - 数据读写
   - 云函数调用
   - 文件上传

#### 错误排查
查看控制台输出，常见错误:
- `cloud function not found`: 云函数未部署
- `permission denied`: 权限设置错误
- `network error`: 网络问题

### 2. 体验版发布

#### 上传代码
1. 在微信开发者工具中点击"上传"
2. 填写版本号（如 `1.0.0`）
3. 填写项目备注
4. 点击"上传"

#### 提交审核
1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入"版本管理"
3. 在"开发版本"中选择刚才上传的版本
4. 点击"提交审核"
5. 填写审核信息:
   - 功能页面截图
   - 功能描述
   - 隐私协议链接
6. 提交

#### 体验版测试
1. 在"版本管理" -> "体验版"
2. 点击"生成体验版"
3. 复制体验版二维码
4. 分享给测试人员扫码体验

### 3. 正式版发布

#### 审核通过后发布
1. 审核通过后，在"审核版本"中点击"发布"
2. 确认发布信息
3. 点击"发布"

#### 版本管理
- 遵循语义化版本规范: `主版本.次版本.修订号`
  - `1.0.0` -> `1.0.1` (bug修复)
  - `1.0.0` -> `1.1.0` (新功能)
  - `1.0.0` -> `2.0.0` (重大更新)

#### 灰度发布（可选）
1. 发布时选择"全量发布"或"灰度发布"
2. 灰度发布可选择 10% / 30% / 50% 的用户
3. 观察数据后逐步扩大范围

## 域名配置

### 1. 服务器域名配置

#### 必需域名
登录微信公众平台 -> 开发 -> 开发管理 -> 开发设置 -> 服务器域名

**添加以下域名**:
- **request合法域名**: 无（使用云开发无需配置）
- **socket合法域名**: 无
- **uploadFile合法域名**: 无
- **downloadFile合法域名**: 无
- **udp合法域名**: 无

**说明**: 使用云开发时，无需配置服务器域名，云开发会自动处理。

### 2. 业务域名配置（可选）

如果小程序需要内嵌 H5 页面，需要配置业务域名:
1. 进入"开发设置" -> "业务域名"
2. 点击"添加域名"
3. 输入域名（如 `https://your-domain.com`）
4. 上传校验文件到服务器根目录
5. 保存

### 3. web-view 域名配置（可选）

如果使用 `<web-view>` 组件:
1. 进入"开发设置" -> "web-view"
2. 添加业务域名

## 常见问题

### Q1: 云函数调用失败，提示 "cloud function not found"
**原因**: 云函数未部署或部署失败

**解决方案**:
1. 检查云函数是否已部署
2. 查看云函数部署日志
3. 重新部署云函数
4. 检查云函数名称是否正确

### Q2: 数据库权限错误 "permission denied"
**原因**: 数据库权限配置不当

**解决方案**:
1. 检查数据库权限设置
2. 确保 `userId` 与 `auth.openid` 匹配
3. 测试环境可临时设置为"所有用户可读写"

### Q3: 云存储上传失败
**原因**: 权限或容量限制

**解决方案**:
1. 检查云存储权限设置
2. 检查存储容量是否超限
3. 确认文件大小符合限制
4. 检查文件格式是否允许

### Q4: 小程序启动慢
**原因**: 代码包过大或资源加载慢

**解决方案**:
1. 使用分包加载
2. 压缩图片资源
3. 使用 CDN 加速
4. 开启缓存策略

### Q5: 云函数执行超时
**原因**: 处理逻辑复杂或数据量大

**解决方案**:
1. 增加云函数超时时间
2. 增加云函数内存配置
3. 优化处理逻辑
4. 使用异步处理

### Q6: 数据库查询慢
**原因**: 缺少索引或数据量大

**解决方案**:
1. 创建合适的索引
2. 使用分页查询
3. 限制返回字段
4. 考虑数据归档

### Q7: 审核被拒
**原因**: 功能不合规或资料不全

**解决方案**:
1. 仔细阅读审核规范
2. 准备完整的功能截图
3. 完善隐私协议
4. 确保功能描述准确

### Q8: 用户数据丢失
**原因**: 未正确保存或云函数错误

**解决方案**:
1. 检查 `app.persist()` 是否正常调用
2. 查看云函数执行日志
3. 启用数据库备份
4. 考虑使用本地缓存兜底

### Q9: 云开发费用过高
**原因**: 调用次数超出免费额度

**解决方案**:
1. 优化数据库查询
2. 减少云函数调用次数
3. 使用缓存减少重复请求
4. 定期清理无用数据

### Q10: 多人协作开发冲突
**原因**: 云开发环境共用导致

**解决方案**:
1. 每个开发者使用独立的云开发环境
2. 使用 Git 管理代码版本
3. 制定开发规范
4. 使用环境变量区分环境
